## [v0.1.0] - 2026-04-20

### Highlights

- Initial release of AX Spec - Agent Experience specification and CLI tools for building agent-friendly APIs
- Spectral-compatible linting rules enforcing DIRECT principles across three compliance levels
- CLI with lint, enrich, and gen commands for the complete AX workflow

### Added

- `ax-spec lint` command wrapping vacuum for OpenAPI validation against AX rules ([`29a0e49`](https://github.com/plexusone/ax-spec/commit/29a0e49))
- `ax-spec enrich` command for adding `x-ax-*` extensions via inference and API discovery ([`29a0e49`](https://github.com/plexusone/ax-spec/commit/29a0e49))
- `ax-spec gen` command for generating Go code (errors, retry, capabilities, validation) from enriched specs ([`29a0e49`](https://github.com/plexusone/ax-spec/commit/29a0e49))
- Spectral ruleset (`ax-openapi.json`) with rules for all DIRECT principles ([`32a6879`](https://github.com/plexusone/ax-spec/commit/32a6879))
- Three compliance level profiles: L1 Structured, L2 Deterministic, L3 Agent-Ready ([`32a6879`](https://github.com/plexusone/ax-spec/commit/32a6879))
- JSON Schema for `x-ax-*` OpenAPI extensions ([`5d48e8d`](https://github.com/plexusone/ax-spec/commit/5d48e8d))
- `pkg/gen` package for extracting AX metadata and generating typed Go code ([`b2795db`](https://github.com/plexusone/ax-spec/commit/b2795db))
- `pkg/enrich` package for inferring `x-ax-*` extensions from OpenAPI structure ([`b2795db`](https://github.com/plexusone/ax-spec/commit/b2795db))
