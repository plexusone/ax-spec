# v0.1.0

**Release Date:** 2026-04-20

Initial release of AX Spec - Agent Experience specification and CLI tools for building agent-friendly APIs.

## Highlights

- **AX Spec CLI** - Three commands for the complete AX workflow: `lint`, `enrich`, `gen`
- **Spectral Rulesets** - Linting rules enforcing DIRECT principles across three compliance levels
- **Code Generation** - Generate typed Go code from enriched OpenAPI specs

## What is AX?

Agent Experience (AX) is a set of principles and specifications for building APIs that autonomous AI agents can consume reliably. Traditional APIs are designed for human developers who can interpret documentation and debug errors manually. AX-compliant APIs provide:

- **Machine-readable metadata** for capabilities, errors, and retry policies
- **Explicit constraints** declared in the specification
- **Structured errors** with actionable recovery suggestions

## CLI Commands

### `ax-spec lint`

Check OpenAPI specs against AX rules using vacuum:

```bash
ax-spec lint openapi.yaml --level l2
```

### `ax-spec enrich`

Add `x-ax-*` extensions to OpenAPI specs via inference:

```bash
ax-spec enrich openapi.yaml -o openapi-ax.yaml
```

Inference rules:

- `x-ax-capabilities` derived from operationId (e.g., `createPayment` → `create_payment`)
- `x-ax-retryable` set to `true` for GET/HEAD/OPTIONS, `false` for mutations
- `x-ax-required-fields` copied from requestBody schema

### `ax-spec gen`

Generate Go code from enriched specs:

```bash
ax-spec gen openapi-ax.yaml -o pkg/ax --package ax
```

Generated files:

| File | Purpose |
|------|---------|
| `errors.go` | Error code constants and metadata |
| `retry.go` | Retry policy mappings |
| `capabilities.go` | Capability constants and operation mappings |
| `validation.go` | Required field validators |

## Compliance Levels

| Level | Name | Key Requirements |
|-------|------|------------------|
| **L1** | Structured | Valid OpenAPI, explicit types, operationIds |
| **L2** | Deterministic | Strict schemas, `additionalProperties: false` |
| **L3** | Agent-Ready | Full `x-ax-*` extensions, error codes, capabilities |

## x-ax-* Extensions

| Extension | Type | Description |
|-----------|------|-------------|
| `x-ax-required-fields` | `string[]` | Explicit required input fields |
| `x-ax-capabilities` | `string[]` | Semantic capabilities |
| `x-ax-error-code` | `string` | Machine-readable error code |
| `x-ax-error-suggestion` | `string` | Actionable fix suggestion |
| `x-ax-retryable` | `boolean` | Safe to retry? |
| `x-ax-idempotent` | `boolean` | Repeated calls safe? |
| `x-ax-sandboxable` | `boolean` | Safe in test environment? |
